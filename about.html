dns-witness · about & watchlist

A verifiable, tamper-evident record of how a set of watched organizations' internet infrastructure behaves over time.  ← live report

What this records (and what it doesn't)

This is not a reputation database for judging unknown domains. It is the opposite: it proactively watches a known list of organizations and records their DNS and certificate infrastructure over time — where their records resolve, under whose networks (ASN) and in which countries, their DNSSEC validation status, and new subdomains appearing in Certificate Transparency.

Every observation is cryptographically signed and hash-chained; the log head is anchored to the Bitcoin blockchain; and independent nodes can witness it — so the history can be independently verified as un-altered and un-backdated. The purpose is change detection and verifiable evidence: noticing when a watched entity's infrastructure shifts (moves jurisdiction, a DNSSEC signature breaks, a new subdomain appears) and being able to prove what was true, and when.

The signals

  • ASN / country — whose network and which jurisdiction an IP sits in; a shift is a key signal.
  • DNSSEC — secure / insecure / bogus; a signed domain dropping to bogus alongside an IP change is a high-confidence hijack indicator.
  • Vantage — the same name resolved from different viewpoints (resolver / EDNS Client Subnet); disagreement across vantages reveals geofencing.
  • CT — new certificates in Certificate Transparency reveal new subdomains / infrastructure as they appear.
